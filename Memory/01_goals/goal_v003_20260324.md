## North Star

OpenMnemo 应从“AI 对话记忆底座”进一步明确升级为：

**一个面向自然语言创作时代的开源创造资产内核。**

它负责把用户在自然语言驱动的创作活动中沉淀出来的三类核心资产统一收口到 Git/Gitea 体系中：

- 创作结果
- 创作过程
- 创作能力

并将这些资产组织为：

- 可版本化
- 可检索
- 可回溯
- 可复用
- 可演化
- 可被任意上层产品再次调用

的长期资产体系。

## Core Definition

OpenMnemo 开源核心的职责不是直接做成某一个具体产品，而是提供一套稳定的资产内核：

- 统一资产模型：`ProductAsset / MemoryAsset / SkillAsset`
- Git/Gitea 原生存储与版本化
- 记忆树与探针式检索
- FTS + vector + graph + tree/probe 的统一检索层
- 统一导入、提炼、回写能力
- 最小运行时接口，供上层产品调用

## Product Position

OpenMnemo 与 deer-flow 一类系统的关系，定位为：

- OpenMnemo：创造资产内核
- deer-flow 类形态：创造执行层或上层产品形态

OpenMnemo 未来可以衍生出多种上层产品，但开源核心本身不等于其中任意一种产品。

## Open Source Core Boundary

### Must Include

- 资产对象模型与关系模型
- 对话、代码、文档、工作流结果的统一导入
- Git/Gitea 回写与版本化
- 记忆树节点生成、收敛、探针展开
- FTS / vector / graph / probe 统一检索接口
- 技能对象化与来源追踪
- 面向上层产品的最小数据层与运行时接口

### Must Not Rush Into Core

- 完整 deer-flow 式 super agent harness
- 重型多 agent 编排 UI
- 企业级权限、审计、计费
- 插件市场或 skill 市场
- SaaS 托管形态
- 复杂商业授权体系

## Near-term Direction

当前主线应在已有数据层能力基础上继续推进，但目标要从“记忆对象与检索收口”升级为“创造资产内核收口”：

1. 让现有 `memory_unit / source_asset / archive_anchor` 与三类资产模型对齐
2. 把“记忆树 + 探针检索”升为正式导航层，而不是仅保留为检索策略设想
3. 明确 `ProductAsset / MemoryAsset / SkillAsset` 的最小对象结构与关系
4. 先以“资产投影/聚合层”承接当前 `DataLayerAPI` 与运行时对象，而不是立即用新资产模型替换 `session / memory_unit / source_asset / archive_anchor`
5. 让上层 AI Chat、workflow、agent、workspace 未来都通过该内核回写资产

## Non-goals For Current Stage

- 不在当前阶段把 OpenMnemo 直接做成完整 deer-flow 替代品
- 不在当前阶段优先投入复杂 UI 工作台
- 不在当前阶段优先投入企业协作和商业闭环
- 不在当前阶段把开源核心与某一具体上层应用强绑定

## Success Signal

当 OpenMnemo 开源核心能够稳定回答以下问题时，说明本轮定位升级成立：

- 一个项目最终产出了哪些产品资产？
- 这些资产是通过哪些对话、决策、约束和里程碑逐步形成的？
- 这些过程最终沉淀出了哪些可复用技能？
- 上层任意产品是否都能重新调用这些资产，而不破坏用户对数据的拥有权？
