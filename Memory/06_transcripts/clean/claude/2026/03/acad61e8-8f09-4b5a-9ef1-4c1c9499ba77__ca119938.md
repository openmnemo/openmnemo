## OpenMnemo 是什么

OpenMnemo 是一个跨平台 AI 对话记忆管理器。它自动发现、导入、版本控制你和 AI 编程助手（Claude Code、Codex、Gemini CLI、豆包）的对话记录，然后生成一个可搜索的 HTML 仪表盘。

核心流程：**发现对话 → 解析 → 去重 → 导入到 Memory/ → 建立 SQLite 全文索引 → 生成静态 HTML 报告 → 可选部署到 GitHub Pages + Webhook 通知**

Monorepo 5 个包：types（类型定义）、core（解析/导入/索引/召回）、sync（心跳守护进程/配置/锁/告警）、report（HTML 报告生成）、cli（统一命令行）。当前版本 v0.4.1，已发布到 npm。

## 多渠道

OpenMnemo 目前已经有 Webhook 通知能力（飞书/Slack/Discord/Telegram），但只是单向推送。EvoScientist 做的是双向交互——在 Telegram/飞书里直接跟 agent 对话。

对 OpenMnemo 来说，借鉴的价值点不是"在飞书里聊天"，而是：

- **多渠道查询**：在飞书/Telegram 里搜索历史对话、查看最新 session 摘要、触发 recall
- **通知增强**：当前 webhook 只发"有新 session 导入"，可以加上摘要内容、关键发现、异常告警的富文本卡片

实现上不复杂。你已经有 `sendWebhook()` 基础设施，扩展成接收消息 → 调用 search/recall → 回复结果就行。不过这会把 OpenMnemo 从纯 CLI 工具变成一个需要常驻运行的服务，架构复杂度会上一个台阶。

**建议**：先不急着做双向交互，优先把 webhook 通知做得更丰富（带摘要卡片），这个 ROI 最高。

## 持久化记忆

这个领域目前几个做法：

| 项目 | 方式 | 优劣 |
|------|------|------|
| EvoScientist (EvoMemory) | LangChain checkpointer，agent state 序列化 | 绑定 LangChain 生态，跨 agent 不通用 |
| Mem0 | 向量数据库 + LLM 提取记忆片段 | 语义检索强，但记忆质量依赖 LLM 提取准确度 |
| Letta (MemGPT) | 分层记忆（core/recall/archival），LLM 自主管理 | 最接近人类记忆模型，但开销大 |
| Claude Code | `CLAUDE.md` + 项目记忆文件 | 简单直接，但纯文本无语义检索 |
| **OpenMnemo** | Git + SQLite FTS + clean markdown | 可审计、可版本控制、跨客户端，但目前只索引元数据 |

说实话，**没有谁做得特别好**。EvoMemory 解决的是单个 agent 的 session 连续性，Mem0 解决的是语义记忆检索，Letta 最有野心但太重。

OpenMnemo 的独特优势是：**跨客户端 + Git 版本控制 + 可审计**。这是其他方案都没有的。你的短板是搜索只覆盖元数据，不搜索对话内容本身。

**如果要增强 OpenMnemo 的记忆能力，最直接的一步是把 FTS 索引扩展到对话内容**——把 clean markdown 的消息文本也写入 SQLite FTS 表。这样用户就能搜到"上周我跟 Claude 讨论过的那个 authentication bug"，而不只是搜标题和分支名。
