## CLI Commands & Subcommands

### 1. **init** (alias: `i`)
**File:** `src/cmd-init.ts`
- Initialize a MemoryTree workspace in a repository
- Options: `--root`, `--project-name`, `--goal-summary`, `--locale`, `--date`, `--time`, `--force`
- Validates: root exists, no existing AGENTS.md, no external policy files
- Creates: Memory directories, goal/todo/chat files, AGENTS.md template
- Returns: exit code 0/1

### 2. **upgrade** (alias: `up`)
**File:** `src/cmd-upgrade.ts`
- Upgrade repo to MemoryTree without overwriting existing policy
- Options: `--root`, `--project-name`, `--goal-summary`, `--locale`, `--date`, `--time`, `--format`
- Detects state: not-installed → partial → installed
- Preserves existing AGENTS.md if complete (9 signal checks)
- Returns: JSON or text output with state transitions

### 3. **import** (alias: `imp`)
**File:** `src/cmd-import.ts`
- Import one transcript file into MemoryTree archives
- Options: `--source` (required), `--root`, `--client`, `--project-name`, `--global-root`, `--raw-upload-permission`, `--format`
- Validates: root/source exist, transcript has content
- Infers client type (codex, claude, gemini, doubao)
- Outputs: manifest fields (session_id, message_count, paths)

### 4. **discover** (alias: `d`)
**File:** `src/cmd-discover.ts`
- Discover and import local AI transcripts
- Options: `--root`, `--client` (all/codex/claude/gemini/doubao), `--scope` (current-project/all-projects), `--project-name`, `--global-root`, `--raw-upload-permission`, `--limit`, `--format`
- Scans all client source directories
- Filters by scope (current repo vs global)
- Returns: counts, import results, skipped items

### 5. **locale** (alias: `l`)
**File:** `src/cmd-locale.ts`

- Options: `--root`, `--locale` (auto/en/zh-cn), `--format`

## Key Design Patterns

1. **Options Interfaces** — Each command has typed `*Options` interface
2. **Exit Codes** — 0 = success, 1 = error
3. **Dual Output** — All commands support `--format text|json`
4. **Immutable Config** — Locale/scaffold functions return new objects
5. **Path Normalization** — POSIX paths via `toPosixPath()` from @openmnemo/core
6. **Error Handling** — Explicit validation at boundaries, stderr for errors
7. **Platform Abstraction** — Daemon uses platform detection (linux/darwin/win32)
8. **Security** — Report server validates path containment, rejects traversal
