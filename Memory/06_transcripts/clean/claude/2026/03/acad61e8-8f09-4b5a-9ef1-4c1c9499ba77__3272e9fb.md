## @openmnemo/report Package - Complete Overview

### Package Information
- **Version**: 0.4.1
- **Type**: ESM TypeScript (NodeNext)
- **Entry Point**: `src/index.ts` → exports `buildReport` + `BuildReportOptions`
- **Build**: tsup with ESM + declaration files
- **Tests**: Vitest with 15s timeout, ~602 tests across all modules

### Core Architecture

**Main Orchestrator: `build.ts`**
- `buildReport(options)` - Main entry point that coordinates entire report generation
- Loads manifests from `Memory/06_transcripts/manifests/`
- Processes transcripts in parallel with bounded semaphore (max 5 concurrent AI calls)
- Generates 9 output pages + RSS feed
- Handles GitHub Pages deployment and webhook notifications
- Ensures `.gitignore` entry for output directory

**Key Data Flow:**
1. Load all manifest JSONs recursively
2. Build link graph from clean markdown references (`[[session_id]]`)
3. Parse clean markdown messages for each session
4. Extract tool summaries from raw JSONL files
5. Generate AI summaries (cached by SHA256) in parallel
6. Extract AI tags (3-5 keywords) from summaries
7. Render all pages and write to output directory
8. Deploy to GitHub Pages (optional)
9. Send webhook notifications (optional)

### Type System (`types.ts`)
- `ReportStats` - aggregated metrics (sessions, messages, tool events, date ranges, client/day/week/tool counts)
- `SummaryCache` - SHA256-based cache for AI summaries
- `LinkGraph` - bidirectional session references (backlinks/forwardLinks)
- `SearchIndexEntry` - indexed session metadata for full-text search
- `BuildReportOptions` - configuration with locale, deployment, webhook settings

### AI Integration

**Summarization (`summarize.ts`)**
- Generates 2-3 sentence summaries of sessions using Claude
- Semaphore limits to 5 concurrent API calls
- SHA256-based caching in `Memory/.report-cache/`
- Graceful fallback: returns empty string on API errors or missing SDK
- Prompt: first 30 messages (500 chars each), asks for accomplishments/problems

**Tag Extraction (`tags.ts`)**
- Extracts 3-5 keyword tags from summaries using Claude
- JSON-only response format
- Cached per SHA256
- Returns empty array on any error (never crashes build)

### Statistics & Analysis (`stats.ts`)

- `buildDayBuckets()` - 'YYYY-MM-DD' → session count
