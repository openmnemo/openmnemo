**[严重度: CRITICAL]** `src/report/deploy/github-pages.ts:39,50,52,62` — 命令注入（Command Injection）

`execSafe` 函数将 `branch` 和 `worktreePath`/`relOutput` 通过模板字符串直接拼接进 shell 命令，但 `execSync` 在传入字符串时会经过 shell 解析（默认调用 `/bin/sh -c`）。

```typescript
execSafe(repoRoot, `git ls-remote --exit-code --heads origin ${branch}`)
execSafe(repoRoot, `git worktree add --orphan -b ${branch} "${worktreePath}"`)
execSafe(repoRoot, `git subtree push --prefix "${relOutput}" origin ${branch}`)
```

`branch` 来自 `config.toml`（用户可控），若包含 `;rm -rf /` 或反引号等字符，可在本机执行任意命令。`worktreePath` 虽来自 `join(repoRoot, ...)` 但其上游 `repoRoot` 同样来自用户配置。

**建议：**
1. 使用 `execSync` 的 **数组参数形式**（spawn 语义）或改用 `child_process.spawnSync` 配合 `argv[]` 数组，彻底绕开 shell 解析。
2. 至少对 `branch` 做严格白名单校验（仅允许 `[a-zA-Z0-9._/-]`），并在 `config.ts` 的 `parseRaw` 阶段执行，不要在 deploy 层才验证。

---

**[严重度: HIGH]** `src/report/render/layout.ts:87-88` — 储存型 XSS（popover innerHTML 未转义）

```typescript
popover.innerHTML = (meta ? '<div class="popover-meta">' + meta + '</div>' : '') +
  (summary ? '<div class="popover-summary">' + summary + '</div>' : '');
```

`meta` 和 `summary` 来自 HTML 元素的 `data-summary` / `data-meta` 属性。这两个属性由服务端渲染时写入，但若上游（transcript 标题、AI 摘要内容）含有 HTML 特殊字符，即便服务端使用了 `escHtml()` 正确编码属性值，浏览器在 `getAttribute()` 取值时会**自动解码**，使得 `popover.innerHTML` 直接解析为 HTML。攻击者若能控制 session 标题（例如 `<img src=x onerror=alert(1)>`），该字符串会经 `escHtml` 写入属性，但在 `getAttribute + innerHTML` 路径下仍然执行。

**建议：** 将 `popover.innerHTML = ...` 改为使用 `textContent` 分别设置子节点，或使用 `innerText`，完全避免 HTML 解析。

---

**[严重度: HIGH]** `src/report/deploy/webhook.ts:130` — SSRF（缺少目标 URL 白名单校验）

```typescript
const transport = url.protocol === 'https:' ? https : http
```

函数允许 `http:` 协议（明文），且 `url` 的 hostname 无任何限制，可以指向：
- 内网地址（`169.254.169.254` 云元数据服务、`10.x.x.x`、`localhost`）
- `file:` 之外的任何协议（`new URL('http://...')` 不会抛出）

config 层的 `webhook_url` 解析仅校验了类型为 `string`，未做任何 URL 格式或 host 验证。

**建议：**
1. 强制 `https:` 协议（拒绝 `http:`）。
2. 解析后检查 `url.hostname` 不属于 RFC 1918 私网范围，可用正则或 `ipaddr.js` 库。
3. 或在 `config.ts` 的 `parseRaw` 阶段就做 URL 格式校验，失败则回退为空字符串（等同于禁用）。

**[严重度: MEDIUM]** `src/report/build.ts:113-126` — `Promise.all` 缺少单任务错误隔离

```typescript
summaryPromises.push(
  (async () => {
    const summary = await getSummary(...)  // 可能抛出
    ...
    writeFileSync(outPath, html, 'utf-8')  // 可能抛出
  })()
)
await Promise.all(summaryPromises)         // 一个失败 → 全部中断
```

`getSummary` 内部有 semaphore 和 try/catch，但 `writeFileSync` 没有。若磁盘空间不足或权限错误，**单个 transcript 的写入失败会通过 `Promise.all` 直接 reject 并中止整个报告构建**，即使已经渲染了数百个页面也会全部丢失。

**建议：** 在每个 `async` 任务的顶层加 `try/catch`，将单条错误转为日志警告并 `resolve()`，使 `Promise.all` 的语义变为"尽力完成所有任务"而非"全或无"。

**[严重度: MEDIUM]** `src/report/deploy/github-pages.ts:78-85` — `makeRelative` 对路径遍历无防护

```typescript
function makeRelative(base: string, target: string): string {
  const b = base.replace(/\\/g, '/').replace(/\/$/, '')
  const t = target.replace(/\\/g, '/').replace(/\/$/, '')
  if (t.startsWith(b + '/')) {
    return t.slice(b.length + 1)
  }
  return t  // 回退：直接返回绝对路径
}
```

当 `outputDir` 不在 `repoRoot` 下时，函数返回 `target` 的完整绝对路径（第 85 行）。此时 `git subtree push --prefix "/absolute/path"` 会以绝对路径作为 prefix，git 通常会报错，但这个静默回退隐藏了配置错误，应当明确报错而非悄悄传入异常值。

**[严重度: LOW]** `src/report/render/graph.ts:350-355` — 无限动画 loop，无退出机制

```typescript
function loop() {
  tick();
  draw();

```

`tick()` 内部在 `ALPHA < ALPHA_MIN` 时 `return`（不更新），但 `draw()` 仍然每帧调用。稳定后每帧无效重绘，在有数百节点的页面上（`MAX_NODES = 500`）会持续消耗 GPU/CPU。**建议**在 `ALPHA < ALPHA_MIN` 且无拖拽/悬停时停止 `requestAnimationFrame`，事件触发时再恢复。
