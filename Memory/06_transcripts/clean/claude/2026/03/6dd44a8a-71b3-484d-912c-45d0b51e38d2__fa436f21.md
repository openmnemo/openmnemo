### 9. Heartbeat.ts - Where buildReport() is Called

**In `processProject()` function (lines 135-149):**

```typescript
export async function processProject(
  config: Config,
  projectPath: string,
  projectName: string
): Promise<void> {
  // ... import transcripts ...
  
  if (config.generate_report) {
    try {
      const { buildReport } = await import('../report/build.js')
      await buildReport({
        root: projectPath,
        output: join(projectPath, 'Memory', '07_reports'),
        noAi: !process.env['ANTHROPIC_API_KEY'],
        model: config.ai_summary_model,
      })
      logger.info(`[${projectName}] Report generated.`)
    } catch (err: unknown) {
      logger.warn(`[${projectName}] Report generation failed: ${String(err)}`)
      // Never propagate — report failure must not abort heartbeat cycle
    }
  }
}
```

**Key context:**
- Called only if `config.generate_report` is `true`
- Report output goes to `Memory/07_reports/`
- Wrapped in try-catch so failures don't abort heartbeat
- Uses config's `ai_summary_model` field
- Disables AI summaries if `ANTHROPIC_API_KEY` is absent

**Broader heartbeat flow:**
1. `main()` acquires lock
2. Calls `runHeartbeat(config)` which iterates over `config.projects`
3. For each project, calls `processProject()`:
   - Discovers transcripts from source files (codex, claude, gemini)
   - Parses and imports new transcripts
   - Scans for sensitive patterns
   - Commits and pushes to git
   - **Generates report** (if enabled)
