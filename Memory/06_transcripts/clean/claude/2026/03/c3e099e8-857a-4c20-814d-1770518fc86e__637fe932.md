**什么是 npm token**

npm token 是 npm（Node.js 包管理平台）颁发的访问凭证，用于在命令行或 CI/CD 流水线中以你的账户身份执行操作，最常见的用途是发布包（`npm publish`）。

它的工作方式：
- 你登录 npm 网站，在账户设置里生成一个 token（一串随机字符串）
- 把这个 token 配置在 CI/CD（比如 GitHub Actions secrets 里的 `NPM_TOKEN`）
- 流水线运行 `npm publish` 时，用这个 token 代替密码进行身份验证

**为什么之前要求轮换？**

因为 token 曾经在 session 对话记录里出现过（比如命令输出或配置内容被记录进了对话文件里）。一旦 token 泄露，任何拿到它的人都可以：
- 以你的名义发布新版本的包
- 覆盖已有版本（如果是 automation token）
- 污染供应链

**轮换的操作**：
1. 去 [npmjs.com](https://www.npmjs.com) → Account Settings → Access Tokens → 删除旧 token，生成新 token
2. 在 GitHub 仓库 Settings → Secrets → 把 `NPM_TOKEN` 的值替换为新 token

如果你已经做了，在 todo 里把那行勾掉就好。

## 一、向量数据库选型

**sqlite-vec** 是最适合 OpenMnemo 的选择，原因是零额外进程、数据文件可以 git 管理、和现有 db.ts 无缝集成。

```
npm install sqlite-vec
```

原理：给现有 SQLite 加一个扩展，新增一种列类型 `float32[N]`，支持 KNN 近邻查询。

```sql
-- 建向量表
CREATE VIRTUAL TABLE vec_sessions USING vec0(
  session_id TEXT,
  embedding float32[1536]   -- OpenAI ada-002 维度
);

-- 插入
INSERT INTO vec_sessions VALUES ('sess_abc', '[0.1, 0.2, ...]');

-- 检索最近邻 top-5
SELECT session_id, distance
FROM vec_sessions
WHERE embedding MATCH '[0.3, 0.1, ...]'
ORDER BY distance LIMIT 5;
```

现有 FTS4 表做关键词匹配，vec_sessions 做语义匹配，两个结果合并用 RRF（Reciprocal Rank Fusion）算法重排序，这就是"混合检索"。

## 三、LLM 提取层详解

这是整个记忆体系的"大脑"，在 heartbeat 同步时触发。

```
新对话导入
    ↓
Haiku 读对话全文（便宜，~$0.0001/次）
    ↓
提取三类信息：
  1. 关键事实   → "决定用 sqlite-vec 做向量检索"
  2. 实体       → [SQLite, openmnemo, FTS, 向量检索]
  3. 实体关系   → SQLite --用于--> openmnemo
    ↓
┌──────────────────────────────────────┐
│  sqlite-vec: 对"关键事实"做 embedding  │  ← 语义检索用
│  Kuzu: 写入实体节点 + 关系边           │  ← 关系追踪用
│  SQLite FTS: 写入对话全文              │  ← 关键词检索用
└──────────────────────────────────────┘
```

Haiku 的提取 prompt 大概长这样：

```
从以下对话中提取：
1. 关键决策/结论（1-3句话）
2. 涉及的技术/项目/人名实体
3. 实体之间的关系（三元组格式：主体-关系-客体）

对话内容：
{conversation_text}

返回 JSON：
```

```
    ↓
```

## 一、Mem0 的图 Schema 核心发现

Mem0 的图 schema **不是预定义的**，而是 LLM 动态提取的，默认只有一个示例模式：


## 二、OpenMnemo 图 Schema 设计（可升级版）

### 节点类型

```
Person        → 用户/协作者
Project       → 代码仓库/项目
Technology    → 技术栈/工具/框架
Concept       → 抽象概念/决策/方案
Session       → 单次对话
Commit        → Git commit
```

### 关系类型

```
(Person)-[:DISCUSSED]->(Concept)          # 张三讨论了向量检索
(Person)-[:WORKED_ON]->(Project)          # 张三在做 openmnemo
(Session)-[:CONTAINS]->(Concept)          # 这次对话包含了 sqlite-vec 方案
(Session)-[:RELATED_TO]->(Commit)         # 对话关联了某个 commit
(Concept)-[:DEPENDS_ON]->(Technology)    # 向量检索依赖 sqlite-vec
(Concept)-[:EVOLVED_FROM]->(Concept)     # 方案 v2 从方案 v1 演化而来
(Project)-[:USES]->(Technology)          # openmnemo 使用 TypeScript
```

### 查询示例

```cypher
-- "我在 openmnemo 项目里讨论过哪些技术方案"
MATCH (p:Person {name: 'user'})-[:DISCUSSED]->(c:Concept)
      -[:CONTAINS]-(s:Session)-[:RELATED_TO]->(proj:Project {name: 'openmnemo'})
RETURN c.name, s.id, s.title

-- "sqlite-vec 这个技术在哪些 session 里被提到过"
MATCH (t:Technology {name: 'sqlite-vec'})<-[:DEPENDS_ON]-(c:Concept)
      <-[:CONTAINS]-(s:Session)
RETURN s.id, s.title, c.name
```

## 三、可升级的存储适配器架构

这是关键——**用适配器模式隔离存储实现**，Phase 0 用嵌入式，Phase 1+ 无缝换成 Mem0 兼容后端：

```typescript
// packages/core/src/storage/graph-adapter.ts

export interface GraphAdapter {
  upsertEntity(entity: Entity): Promise<void>
  upsertRelation(relation: Relation): Promise<void>
  query(cypher: string, params?: Record<string, unknown>): Promise<unknown[]>
  close(): Promise<void>
}

export interface VectorAdapter {
  upsert(id: string, embedding: number[], metadata: Record<string, unknown>): Promise<void>
  search(embedding: number[], topK: number): Promise<VectorResult[]>
  close(): Promise<void>
}
```

### Phase 0：嵌入式（零依赖）

```typescript
// KuzuGraphAdapter — 单文件，无服务器
import kuzu from 'kuzu'

export class KuzuGraphAdapter implements GraphAdapter {
  // 数据文件存在 Memory/graph/ 目录，可被 git 管理
}

// SqliteVecAdapter — sqlite 扩展
export class SqliteVecAdapter implements VectorAdapter {
  // 复用现有 db.ts 的 SQLite 连接
}
```

### Phase 1+：升级为 Mem0 兼容后端

```typescript
// Neo4jGraphAdapter — 团队版/SaaS
export class Neo4jGraphAdapter implements GraphAdapter { ... }

// QdrantVectorAdapter — Mem0 默认向量库
export class QdrantVectorAdapter implements VectorAdapter { ... }

// Mem0Adapter — 直接对接 Mem0 API（托管版）
export class Mem0Adapter implements GraphAdapter, VectorAdapter { ... }
```

```yaml
# openmnemo.config.yaml
memory:
```
