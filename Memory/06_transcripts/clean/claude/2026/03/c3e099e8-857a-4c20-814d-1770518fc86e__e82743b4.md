### Database Layer (`db.ts`)

**Technology:** sql.js (WebAssembly SQLite) — no native binding required

**Schema:**
- Primary key: `(client, project, session_id, raw_sha256)`
- Data columns: title, started_at, imported_at, cwd, branch, paths, counts
- FTS4 virtual table on: title, cwd, branch (simple tokenizer)

**Key Functions:**
- `upsertSearchIndex(dbPath, manifest)` — Insert/update + rebuild FTS index
- `searchTranscripts(dbPath, query, limit)` — FTS4 MATCH search, ordered by started_at DESC
- `sanitizeFtsQuery(raw)` — Escape special FTS4 characters (keeps letters/digits/apostrophes/spaces)

**Database Location:** `~/.memorytree/transcripts/index/search.sqlite`

### Retrieval Logic (`recall.ts`)

**Main Entry:** `recall(root, projectName, globalRootOverride, activationTime)`

**Flow:**
1. Sync current project: discover + parse + import matching transcripts
2. Find latest session: query SQLite DB (or fallback to JSONL)
3. Load clean markdown content if available
4. Return `RecallResult` with session metadata + content

**Session Matching:**
- Query: `started_at < activationTime` (ordered DESC, limit 20)
- Filter: `cwdMatches(cwd, resolvedRoot)` OR `project === repoSlug`
- CWD matching: exact match or prefix match (case-insensitive, POSIX paths)

**Fallback:** If SQLite unavailable, reads `~/.memorytree/transcripts/index/sessions.jsonl`
