### HIGH PRIORITY FINDINGS

**HIGH: Doubao Client Not Implemented**
- **File:** `packages/core/src/transcript/parse.ts`
- **Lines:** 53-65, 50
- **Issue:** The README advertises support for Doubao transcripts (line 183: "Doubao | Manual (`--source`) | JSON"), and the CLI accepts `--client doubao` (packages/cli/src/main.ts:78, 130), but the actual parsing implementation throws "unsupported transcript client: doubao" (line 64). The CLIENTS constant includes 'doubao' (packages/core/src/transcript/common.ts:15), but there is no `parseDoubaoTranscript()` function. Users will get a confusing error when trying to import Doubao transcripts with `openmnemo import --source file.json --client doubao`.
- **Impact:** Users following the documented workflow for Doubao will experience a runtime error instead of successful import.

**HIGH: PID Command Injection Risk on Windows**
- **File:** `packages/sync/src/lock.ts`
- **Line:** 146
- **Issue:** The Windows process check uses `execFileSync('tasklist', ['/FI', `PID eq ${pid}`, '/NH'])`. While `execFileSync` is safer than `exec`, the `PID eq ${pid}` filter string is user-controlled (it's a PID from the lock file read at line 49-50). A malicious or corrupted lock file could inject tasklist filter expressions. For example, a lock file containing "1 | foo" would create the argument `PID eq 1 | foo`. Although tasklist doesn't interpret pipes in filter expressions, the practice violates security best practices.
- **Impact:** Potential command injection if the lock file is compromised by a malicious actor with file system access.
- **Remediation:** Validate the PID as strictly numeric before use, or use safer Windows API calls.

**HIGH: Missing `main` Field in CLI package.json**
- **File:** `packages/cli/package.json`
- **Lines:** 6-8
- **Issue:** The package has a `bin` field but no `main` field. While the `bin` field correctly points to `./dist/main.js` for the CLI executable, packages that export a `main` field allow users to `require('@openmnemo/cli')` or import it as a library. This is inconsistent with the other packages (core, report, sync, types) which all have `main` and `exports` fields. If users want to programmatically use CLI functions, they cannot.
- **Impact:** Users attempting to import CLI utilities programmatically will fail.
