```typescript
export interface ClientCounts {

  readonly date: string          // "YYYY-MM-DD"
  readonly count: number         // message count that day

export interface WeekBucket {
  readonly weekStart: string     // "YYYY-MM-DD" Monday of the week
  readonly count: number
}

export interface ToolFrequency {
  readonly name: string
  readonly count: number
}

export interface ReportStats {
  readonly totalSessions: number
  readonly totalMessages: number
  readonly totalToolEvents: number
  readonly activeDays: number                    // unique calendar days with ≥1 message
  readonly clientCounts: ClientCounts
  readonly messagesPerDay: readonly DayBucket[]  // sorted ascending, last 365 days
  readonly messagesPerWeek: readonly WeekBucket[] // sorted ascending, last 52 weeks
  readonly topTools: readonly ToolFrequency[]     // top 10, sorted descending
  readonly avgMessagesPerSession: number
  readonly maxMessagesPerSession: number
  readonly dateRange: { readonly first: string; readonly last: string } | null
}

// ─── SummaryCache ────────────────────────────────────────────────────────────
export interface SummaryCache {
  readonly sha256: string        // raw_sha256 of the transcript
  readonly summary: string       // 2-3 sentence AI-generated summary
  readonly model: string         // model id used
  readonly generatedAt: string   // ISO timestamp
}

// ─── LinkGraph ───────────────────────────────────────────────────────────────
// Maps session stem (artifactStem without extension) → list of session stems
// that reference it via [[session-id]] wiki-link syntax.
export type LinkGraph = ReadonlyMap<string, readonly string[]>

// ─── SearchIndexEntry ─────────────────────────────────────────────────────────
export interface SearchIndexEntry {
  readonly id: string            // artifactStem
  readonly title: string
  readonly client: string
  readonly project: string
  readonly startedAt: string
  readonly text: string          // full plain-text content, truncated to 50 kB
  readonly url: string           // relative URL e.g. "transcripts/claude/stem.html"
}

// ─── BuildReportOptions ──────────────────────────────────────────────────────
export interface BuildReportOptions {
  readonly root: string          // repo root (Memory/ lives here)
  readonly output: string        // output dir, default Memory/07_reports
  readonly noAi: boolean
  readonly pdf: boolean          // reserved for future use
}

export interface ServeReportOptions {
  readonly root: string
  readonly port: number
}
```

Key design decisions:
- `messagesPerDay` uses the last 365 days only (beyond that the heatmap becomes unreadable).
- `LinkGraph` is immutable and uses a `ReadonlyMap` to match the immutability coding style requirement.
- `SearchIndexEntry.text` is truncated to 50 kB to keep the embedded JSON in `search.html` from bloating the file beyond reason.

## 3. SVG Chart Designs

### 3a. GitHub Heatmap (`renderHeatmap`)

The implementation scans the last 365 days ending today, organized into 53 ISO weeks × 7 weekdays. Each day cell is a 10×10 `<rect>` with `rx="2"` (rounded corners), spaced 12px apart horizontally and 12px vertically.

Color scale (5 levels, GitHub Dark palette):
- Level 0 (0 messages): `#161b22` (dark background)
- Level 1 (1-2): `#0e4429`
- Level 2 (3-6): `#006d32`
- Level 3 (7-12): `#26a641`
- Level 4 (13+): `#39d353`

SVG dimensions: `width="680" height="112"`. Month labels rendered above the grid at the column where a new month starts. Day-of-week labels (Mon, Wed, Fri) on the left side.

Each rect carries a `<title>` child: `"YYYY-MM-DD: N messages"` for browser tooltip on hover.

```
viewBox="0 0 700 130"
  <text> month labels row (y=14)
  <text> weekday labels col (x=0)
  for each (week, day):
    <rect x={week*12+40} y={day*12+20} width=10 height=10 rx=2
          fill={levelColor(count)} >
      <title>date: N messages</title>
    </rect>
```

### 3b. Client Doughnut (`renderClientDoughnut`)

Pure SVG arc path using `d="M ... A ... Z"` construction. Three segments for codex/claude/gemini.

Dimensions: `viewBox="0 0 200 200"`. Center: `(100,100)`. Outer radius: 80. Inner radius: 50 (creating the doughnut hole).

Color map:
- codex: `#58a6ff`
- claude: `#bc8cff`
- gemini: `#3fb950`

Each arc segment is computed from cumulative angles. The label text (percentage) is placed at the midpoint of each arc at radius 95 (outside the doughnut). A `<text>` in the center shows "Sessions".

Legend rendered below as three `<rect>` color swatches + `<text>` labels.

### 3c. Weekly Line Chart (`renderWeeklyLine`)

Dimensions: `viewBox="0 0 600 120"`. Plot area: 560×80 with 20px left and 20px top padding.

X axis: 52 equidistant points. Y axis: 0 to max count, normalized to 80px height.

Two SVG elements:
1. `<path>` for the filled area: `M x0,bottom L x0,y0 L x1,y1 ... L xN,bottom Z` with `fill="rgba(88,166,255,0.2)"` (GitHub blue at 20% opacity).
2. `<path>` for the line stroke: `M x0,y0 L x1,y1 ...` with `stroke="#58a6ff" stroke-width="1.5" fill="none"`.

No axis ticks for cleanliness. A hover layer is not needed (this is static HTML).

### 3d. Tools Horizontal Bar Chart (`renderToolBar`)

Dimensions: `viewBox="0 0 500 220"`. Renders at most 10 tools.

Each row is 20px tall with 2px gap. Bar color: `#3fb950` (green). Max bar width = 380px (normalized to the top tool's count). Tool name as `<text>` left-aligned at x=0. Count label as `<text>` right of the bar end.

```
for i in 0..9:
  y = i * 22
  barWidth = (tool.count / maxCount) * 380
  <text x=0 y={y+14}>{tool.name}</text>      // left label, 120px wide
  <rect x=125 y={y+2} width={barWidth} height=16 rx=2 fill="#3fb950"/>
  <text x={125+barWidth+4} y={y+14}>{tool.count}</text>
```

### `summarize.ts` Implementation Detail

```typescript
// Model constant
const DEFAULT_SUMMARY_MODEL = 'claude-haiku-4-5-20251001'

// System prompt
const SYSTEM_PROMPT = `You are a technical assistant that summarizes AI coding session transcripts.
Write exactly 2-3 sentences. Focus on what was accomplished, what problems were solved, and any
notable technical decisions. Be specific about tools used and code changed. Do not use filler
phrases like "In this session" or "The user". Write in past tense.`

// User prompt builder
export function buildSummaryPrompt(messages: readonly TranscriptMessage[]): string {
  // Take at most 30 messages (15 user + 15 assistant alternating) to stay well
  // within Haiku's context window and keep API costs minimal.
  const sample = messages.slice(0, 30)
  const lines = sample.map(m => `[${m.role.toUpperCase()}]: ${m.text.slice(0, 500)}`)
  return `Summarize this coding session transcript:\n\n${lines.join('\n\n')}`
}

// API call shape
const response = await client.messages.create({
  model: options?.model ?? DEFAULT_SUMMARY_MODEL,
  max_tokens: 200,
  system: SYSTEM_PROMPT,
  messages: [{ role: 'user', content: buildSummaryPrompt(messages) }],
})
const summary = response.content[0]?.type === 'text' ? response.content[0].text.trim() : ''
```

Key guardrails:
- Check `process.env['ANTHROPIC_API_KEY']` before importing the SDK. If absent, return `''` immediately without requiring the SDK to be installed.
- Wrap the API call in try/catch: on error, log a warning and return `''` (never crash the build).
- The SDK import is a dynamic `await import('@anthropic-ai/sdk')` so the module loads only when needed and does not break builds where the SDK is not installed yet.
- Cache file schema: `{ sha256, summary, model, generatedAt }` written as formatted JSON.

## 6. `src/report/build.ts` Orchestration Flow

```
buildReport(options):
  1. resolvedRoot = resolve(options.root)
  2. outputDir = resolve(options.output)         // Memory/07_reports
  3. cacheDir = join(outputDir, '.cache')
  4. mkdirSync all output subdirs
  5. manifests = loadManifests(resolvedRoot)      // glob Memory/06_transcripts/manifests/**/*.json
  6. stats = computeStats(manifests)
  7. linkGraph = buildLinkGraph(manifests, readCleanContent)
  8. index = buildSearchIndex(manifests, readCleanContent)
  9. knowledgeFiles = loadDirectory(join(resolvedRoot, 'Memory/04_knowledge'), '**/*.md')
  10. goalsFiles = loadDirectory(join(resolvedRoot, 'Memory/01_goals'), '**/*.md')

  // Generate summaries in parallel (max 5 concurrent) for sessions without cache
  11. For each manifest: summary = await getSummary(messages, sha256, cacheDir, {...})

  // Write pages
  12. writeFile(join(outputDir, 'index.html'), htmlShell({...renderDashboard(stats)}))
  13. writeFile(join(outputDir, 'transcripts/index.html'), htmlShell({...renderTranscriptList(manifests)}))
  14. For each manifest:
        parsed = parseTranscript(manifest.client, manifest.global_clean_path)  // or repo_clean_path
        backlinks = linkGraph.get(sessionStem) ?? []
        writeFile(join(outputDir, 'transcripts', client, stem + '.html'), renderTranscript(...))
  15. writeFile(join(outputDir, 'knowledge/index.html'), renderKnowledge(knowledgeFiles))
  16. writeFile(join(outputDir, 'goals/index.html'), renderGoals(goalsFiles))
  17. writeFile(join(outputDir, 'search.html'), renderSearchPage(index))
```

Concurrency for summaries: use a simple semaphore (counter + Promise queue) rather than `p-limit` to avoid adding another dependency.

## 9. Gotchas and Edge Cases

### Manifest loading
`Memory/06_transcripts/manifests/` is partitioned by `client/year/month/`. Use `fast-glob` pattern `Memory/06_transcripts/manifests/**/*.json` — the same glob package is already a dependency. Sort by `started_at` for stable output ordering.

### Clean markdown path resolution
`manifest.repo_clean_path` is a POSIX-relative path (e.g. `Memory/06_transcripts/clean/...`). Use `join(root, manifest.repo_clean_path)` to resolve it. Some manifests may have `repo_mirror_enabled: false` and an empty `repo_clean_path` — in that case fall back to `manifest.global_clean_path`. Always guard with `existsSync` and return `''` on missing files.

### `--no-ai` Commander.js flag gotcha
Commander treats `--no-<name>` as a boolean negation flag. The option declared as `.option('--no-ai', ...)` will populate `opts.ai` as `false` when passed and `true` by default. The `noAi` property must be `!opts.ai`. This matches the existing `--no-<x>` patterns in Commander.

### `@anthropic-ai/sdk` optional dependency
The SDK import must be dynamic (`await import('@anthropic-ai/sdk')`), placed inside a try/catch that returns `''` on `MODULE_NOT_FOUND`. This way the package is truly optional — the build still works even if the user hasn't installed it. Add to `package.json` as a regular dependency (not optional), but the code must defensively handle its absence for the period before npm install is re-run.

### HTML escaping
All user-generated content (titles, session IDs, message text, file paths) that is interpolated into HTML strings must be HTML-escaped. Write a small `escHtml(s: string): string` utility in `render/layout.ts` that handles `& < > " '`. This is the most common XSS source in template-style HTML generation.

### File system on Windows
The project runs on Windows (confirmed by the platform info). All path construction must use `node:path` `join()`, not string concatenation with `/`. The `toPosixPath` utility from `src/utils/path.ts` is already available for converting before writing into HTML href attributes.

### Large transcript sets
`buildReport` must not hold all parsed transcript content in memory simultaneously. Process transcripts in a streaming fashion: parse → summarize → render → write → discard. Keep only manifests (small) in memory throughout. The summary concurrency limit (5 parallel) also limits peak memory.

### `.gitignore` for `Memory/07_reports/`
The requirements specify the output dir should not be committed. The `cmd-report.ts` or `build.ts` should check if `Memory/07_reports` is already in `.gitignore` and append it if not. Alternatively document this as a manual step, since the `init` command scaffolds `.gitignore` already — the safer approach is to add `Memory/07_reports/` to the default gitignore template in `src/project/scaffold.ts`.

### Wiki-link regex
The `[[session-id]]` pattern must use a non-greedy match: `/\[\[([^\]]+?)\]\]/g`. Session stems can contain hyphens and underscores but not `]]`. The regex must not match across newlines.

### Empty / single-data-point charts
- Doughnut with all zeros: render an empty gray ring, no percentage labels.
- Line chart with 1 data point: render a single dot (circle), not a path.
- Heatmap with no data: render all gray cells (still useful as a date grid).
- Bar chart with no tools: render the SVG with a "No tool events recorded" text node.

### `serve` command port conflicts
Wrap the HTTP server creation in a try/catch for `EADDRINUSE`. Print a clear message: `"Port NNNN is in use. Try --port MMMM."` and exit with code 1.

### Report cache invalidation
The cache key is `raw_sha256` (the SHA256 of the raw source file). This means the cache entry is permanently valid as long as the source file has not changed — which is the correct invariant. No TTL is needed.
