#### 3. **lock.ts** — Single-Instance Enforcement
**Purpose:** PID-based lock file to prevent concurrent heartbeat runs

**Key Exports:**
- `lockPath()` — Returns `~/.memorytree/heartbeat.lock`
- `acquireLock()` — Try to acquire lock (returns bool)
- `releaseLock()` — Release lock
- `readLockPid()` — Read PID from lock file (or null)
- `isProcessAlive(pid)` — Check if process running (Unix: `process.kill(pid, 0)`; Windows: `tasklist`)

**Behavior:**
- Uses `O_CREAT | O_EXCL` to prevent TOCTOU races
- Detects stale locks: checks if stored PID is alive
- Reclaims stale locks from dead processes
- Handles corrupt lock files gracefully
- Platform-aware: Unix uses `process.kill()`, Windows uses `tasklist`
