## 一、我们在解决什么问题

### 时代背景

大模型正在从"聊天玩具"变成人类最重要的生产力工具。全球数亿人每天用 Claude、ChatGPT、Gemini、Copilot 工作、学习、创作。

但所有人都面临同一个痛点——

### 核心痛点

**1. 记忆碎片化**
用户在 VS Code 里用 Copilot 写代码、在网页用 ChatGPT 聊方案、在手机用 Gemini 问问题、在飞书里讨论需求。这些对话散落在 N 个工具、N 个平台，无法统一管理，时间一长全部遗忘。

**2. 上下文失控**
长会话越来越长，大模型的注意力机制会让早期内容被"淹没"。平台方做黑盒压缩，用户不知道丢了什么，导致 AI 反复幻觉、反复偏离目标，用户不得不一遍遍重新投喂背景。

**3. 资产归零**
用户花了大量时间和 AI 交流，里面有想法、有决策、有创作过程。但这些全部是平台的数据，不属于用户。平台关闭、账号封禁，一切清零。

### 问题规模

- 全球 AI 对话工具月活：**超过 5 亿用户**
- 每人每天平均发起 AI 对话：**3-10 次**
- 用户能找回一个月前某次对话的概率：**< 5%**

这不是小问题。这是 AI 时代最大的信息损耗。

## 二、我们的解法

**OpenMnemo** —— 人类与 AI 交互的记忆操作系统

核心理念：**一切皆仓库**

```
用户的所有 AI 对话
        ↓
自动捕获、清洗、分类、索引
        ↓
存入用户自己的 Gitea 仓库
  ├── 开源用户：直接用 GitHub（零门槛）
  └── 企业/隐私用户：自部署 Gitea（完全自主）
        ↓
结构化记忆 + 可视化看板 + 随时检索
        ↓
带着完整记忆继续下一次对话
无幻觉 · 不重复投喂 · 资产永久属于用户
```

### 为什么是 Gitea，而不只是 Git

Git 只是版本控制协议。**Gitea** 是完整的代码托管平台——开源、轻量、可自部署，提供仓库管理、Issues、Wiki、Actions、Webhooks、API 等完整能力，是 GitHub 的开源替代品。

OpenMnemo 以 Gitea 为底座，意味着：
- 用户可以用 **GitHub**（最简单，开箱即用）
- 用户可以**自部署 Gitea**（数据完全不离开自己的服务器）
- 两种方式对用户完全透明，功能完全一致
- 未来支持 GitLab、Forgejo 等任何 Gitea 兼容平台

### Gitea 的账号体系 = 天然的记忆分类系统

Gitea 有两种主体：**个人账号** 和 **组织（Organization）**。

```
个人账号 @alice
  ├── 个人仓库（私有）
  │     ├── learning/        ← 学习成长类对话
  │     ├── health/          ← 健康医疗类对话
  │     ├── personal/        ← 个人生活类对话
  │     └── finance/         ← 财务规划类对话
  │
  ├── 加入家庭组织 @alice-family（internal 可见性）
  │     ├── 家人对话记录（彼此可查）
  │     └── 孩子教育（仅家庭成员可见）
  │
  └── 加入公司组织 @my-company（internal 可见性）
        ├── 工作项目对话（团队共享）
        └── 技术方案讨论
```

**组织（Organization）** 可以创建团队（Team），精细控制每个成员对每个仓库的权限（只读 / 读写 / 管理员）。这让"家庭共享记忆"和"工作团队协作"都有原生支持。

**Gitea 的命名空间就是语义分类系统**——不需要额外设计，Gitea 已经有完整的答案。

### 隐私模型

| 可见性 | 含义 | 适用场景 |
|--------|------|---------|
| `private` | 仅自己可见 | 个人隐私记忆（默认） |
| `internal` | 同一 Gitea 实例所有登录用户可见 | 家庭/团队共享内容 |
| `public` | 全网可见 | 知识输出、公开分享 |

**默认隐私设置**：用户在 OpenMnemo 中设置一次全局默认（推荐 `private`），每个仓库可单独覆盖。企业用户自部署 Gitea 后，`internal` 可见性等于"仅本公司员工可见"，完美满足企业隐私需求。

### Gitea 平台能力如何服务 OpenMnemo

| Gitea 功能 | OpenMnemo 用途 |
|-----------|---------------|
| 仓库 + 版本控制 | 所有对话的版本化存储，可回溯任意历史 |
| Issues | 从对话中提取任务，自动创建 Issue 跟踪 |
| Wiki | 知识沉淀区（04_knowledge/ 对应 Wiki） |
| Actions（CI/CD） | 心跳触发自动构建报告、推送 Pages |
| Webhooks | 实时推送通知到飞书/Telegram/Discord/Slack |
| API（兼容 GitHub API） | CLI / 浏览器插件 / App 的统一数据接入层 |
| Gitea Pages | 记忆看板静态部署（与 GitHub Pages 等价） |
| 组织 + 团队 | 家庭/企业的多人协作与权限管理 |
| `internal` 可见性 | 家庭/团队内部共享，不对外暴露 |

### 和现有产品的本质区别

| 维度 | 现有 AI 工具 | OpenMnemo |
|------|------------|-----------|
| 数据归属 | 平台控制 | 用户完全拥有，存在自己的 Gitea/GitHub 仓库 |
| 跨平台 | 各自封闭 | 全平台统一汇聚 |
| 记忆持久性 | 会话结束即消失 | 版本化永久保存，可回溯任意历史 |
| 上下文管理 | 黑盒压缩 | 白盒可控，用户决定加载什么 |
| 资产价值 | 消耗品 | 可检索、可复用、可分享的数字资产 |
| 分类体系 | 平台封闭分类 | Gitea 命名空间：个人仓库 + 组织 + 团队，语义天然清晰 |
| 家庭/团队共享 | 无 | Gitea 组织 + internal 可见性，原生支持 |
| 部署方式 | 绑定平台服务器 | GitHub（开源用户）或自部署 Gitea（企业/隐私用户）|

## 三、产品形态与路线图

### 第一阶段（当前）：CLI + 记忆看板
- 命令行工具，自动捕获 Claude Code / Codex / Gemini CLI 的对话
- 心跳守护进程，5 分钟同步一次，自动 push 到 Gitea/GitHub
- 生成可视化记忆看板，一键部署到 GitHub Pages 或自托管 Gitea Pages
- **目标用户**：重度 AI 编程用户（全球约 500 万人）

### 第二阶段：浏览器插件
- 捕获网页版 AI 对话（Claude.ai / ChatGPT / Gemini Web）
- 用户授权后，自动归档到记忆仓库
- **打通非开发者用户，用户规模跃升 10 倍以上**

### 第三阶段：IM 推送
- 飞书 / Telegram / Discord / Slack 机器人
- 新记忆自动推送通知，附带跳转链接
- 让记忆"主动找到"用户

### 第四阶段：Web 端 + 桌面 App
- 完整的记忆管理界面
- 跨项目检索、知识图谱、AI 续写
- Windows / macOS / Linux 原生客户端（Tauri）

### 第五阶段：手机 App + 生态
- iOS / Android，随时随地查看、继续对话
- 团队协作、仓库模板市场
- 第三方开发者 API

## 七、竞争优势与护城河

**1. 中立性**
OpenMnemo 不绑定任何 AI 厂商、任何平台。这是大厂永远无法做到的——微软要绑 Copilot，OpenAI 要绑 ChatGPT，Google 要绑 Gemini。我们是唯一真正中立的记忆层。

**2. 数据主权**
用户的记忆存在自己的 Gitea/GitHub 仓库，我们只是工具。开源用户用 GitHub 零门槛起步；对数据主权要求更高的企业用户，可以自部署 Gitea，数据完全不经过任何第三方服务器。这解决了最深层的信任问题，也是最强的用户粘性——用户的记忆越积累，迁移成本越高，留存率越高。

**3. Gitea 生态的完整性**
我们不只是存储层，而是完整平台能力的使用者：版本控制、Issues 任务追踪、Wiki 知识库、Actions 自动化、Webhooks 通知、组织协作——这些都是 Gitea/GitHub 已经解决好的问题，我们站在上面构建，而不是重造轮子。

**4. 开源社区**
开源建立信任，社区驱动传播。每个开发者都是种子用户，也是传播节点。

**5. 先发优势**
记忆资产一旦积累，用户不会迁移。第一个被用户信任的记忆平台，将长期保持领先。
