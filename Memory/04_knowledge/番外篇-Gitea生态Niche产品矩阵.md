# 番外篇：以 Gitea 为底座的 Niche 产品矩阵

> 版本：v1.0 · 日期：2026-03-17
> 定位：OpenMnemo 生态扩展方向，各产品共享同一技术底座

---

## 核心思路

OpenMnemo 验证了一个模式：**Gitea 的原生能力（仓库、Issues、PR、Wiki、Org、Actions）+ AI 翻译层**，可以替代某个垂直领域里昂贵、封闭、割裂的现有工具。

这个模式可以复制到任意领域。变的只是领域壳和 AI Prompt，不变的是底座。

---

## Gitea 核心能力映射

| Gitea 能力 | 抽象含义 | 可替代的传统工具 |
|-----------|---------|----------------|
| 仓库 | 任何一个领域空间 | 文件夹、项目空间 |
| 版本控制 | 内容的完整修改历史（时间机器） | 手动备份、快照 |
| Issues | 任务 / 问题追踪 | Trello、Jira、便利贴 |
| PR + Review | 审批 / 协作流 | 邮件往返、审批系统 |
| Wiki | 参考知识库 | Notion、Confluence |
| Org + Team | 多人协作 + 权限控制 | 企业OA、钉钉 |
| Internal 可见性 | 私密共享（家庭 / 团队） | 私有群组 |
| Actions | 自动化触发 | Zapier、n8n |
| Webhooks | 事件推送通知 | 各平台 Bot |
| 自部署 Gitea | 数据主权，隐私敏感场景 | 无可替代 |

---

## 选品判断框架

一个 Niche 是否值得做，看五个维度：

```
✅ 现有工具很烂或很贵（痛点够深）
✅ 内容天然是文本 / Markdown（技术成本低）
✅ 有审批 / 协作流需求（PR 能替代昂贵流程）
✅ 隐私敏感（自部署是竞争壁垒）
✅ 小团队 / 个人能覆盖的市场（niche，不需要打大厂）
```

## Tier 1 — 优先级最高

### 1. 法律文书管理平台

**痛点：** 律所用 Word + 邮件往返审合同，没有版本追踪，没有协作流，数据在第三方服务器。

```
合同文本  → Markdown 文件，完整修改历史 = 法律审计链
PR       → 律师 / 客户逐条审阅、批注、接受或拒绝每处修改
Issues   → 合规问题追踪、条款争议、待处理事项
Wiki     → 合同模板库、法律术语表、案例参考
Actions  → 合同到期自动提醒、签署状态变更通知

AI 增值：
  - 草拟标准合同初稿
  - 识别高风险条款并高亮提示
  - 建议符合当地法规的标准语言
  - 中英文合同双语对照检查
```

**竞争空白：** 现有法律 SaaS（DocuSign、ContractPodAi）价格极高且数据在云端，中小律所和个人用户完全没有好选择。自部署 Gitea 解决律所数据主权顾虑。

---

### 2. 个人 CRM / 人脉管理

**痛点：** 没有真正好用的个人 CRM。现有产品要么太企业化，要么太简陋，隐私数据全在厂商服务器。

```
每个联系人 → 独立 Markdown 文件（背景、关系脉络、聊天摘要、跟进记录）
Issues    → "三个月后跟进 Alice 关于融资的事" / 定期维系提醒
Wiki      → 行业地图、关键人脉关系网络、圈子分析
Org       → 按圈子分组（投资圈、老同学、合作伙伴）

AI 增值：
  - 关系强度评分（最近联系频率 + 互动深度）
  - 自动从对话记录提取关键信息更新联系人档案
  - 跟进时机推荐（"Alice 上次提到她 4 月有融资需求"）
  - 人脉路径分析（"认识 Bob 最短路径是通过 Alice"）
```

**竞争空白：** Monica、Clay 等个人 CRM 工具数据在云端，无法自部署，功能也有限。这个方向有极强的个人用户痛点。

---

### 3. 学术研究 / 论文管理

**痛点：** 研究者用 Zotero 管文献、Word 写笔记、邮件传稿、GitHub 管代码，四个工具完全割裂。

```
每篇论文笔记 → Markdown（摘要、核心论点、引用、自己的批注）
PR          → 导师批改学生论文草稿，逐段 Review
Issues      → 待解决的研究问题、实验设计缺陷、文献缺口
Wiki        → 文献综述、研究方法论、实验日志
Actions     → 论文截止日前自动提醒，arXiv 更新推送

AI 增值：
  - 论文自动摘要（粘贴 PDF，输出结构化笔记）
  - 发现知识库内论文之间的隐性关联
  - 推荐相关未读文献
  - 引用格式检查（APA / MLA / Chicago）
  - 研究空白识别（"这个方向还没人研究过"）
```

**竞争空白：** 高校和研究机构对数据主权要求极高，开源自部署是唯一可接受的方案。现有工具（Notion、Obsidian）没有协作审稿流。

---

### 4. 创意写作 / 小说平台

**痛点：** Scrivener 没有协作，Word 没有版本控制，没有工具支持编辑与作者之间的结构化审稿流程。

```
每章        → 独立 Markdown 文件，完整修改历史可回溯任意版本
PR          → 编辑 / Beta Reader 的修改建议，作者可逐行 accept / reject
Issues      → 情节漏洞、人物弧线待处理问题、读者反馈
Wiki        → 世界观设定、人物卡（外貌/性格/关系）、时间线、地图
Org         → 出版社组织，多作者协同一个系列

AI 增值：
  - 文风一致性检查（"第三章的语气和前两章不一样"）
  - 情节连贯性检查（人物前后矛盾、时间线错误）
  - 根据世界观设定续写
  - 自动生成人物关系图、时间线可视化
  - 多语言版本管理（原著 + 翻译版本对照）
```

**竞争空白：** 网文平台锁定创作者内容，传统写作工具缺乏协作能力，独立作者和小出版社没有好的内容管理方案。

## Tier 2 — 中期机会

### 5. 家庭财务 / 共同记账

```
家庭组织 @family（internal 可见性，家庭成员可见）
每月账单  → YAML / CSV 结构化记录，版本可回溯
Issues   → 大额采购讨论、预算争议、理财计划
Wiki     → 家庭财务规则、保险清单、资产登记、紧急联系
Actions  → 月末自动汇总报告、账单到期提醒

AI 增值：
  - 支出趋势分析与异常检测
  - 储蓄目标进度追踪
  - 年度财务健康报告自动生成
  - 税务相关支出自动分类归档
```

**竞争空白：** 家庭财务软件两头不讨好——个人记账 App 不支持家庭共享，企业财务软件太复杂。Gitea Family Org + Internal 可见性天然适配。

---

### 6. 医疗健康记录

```
自部署 Gitea = 医疗数据完全不经过任何第三方服务器

每次就诊 → Markdown 记录（症状、用药、医嘱、检查结果）
Issues  → 待复查项、慢性病管理任务、用药提醒
Wiki    → 家族病史、药物过敏列表、常用医院/医生
Org     → 家庭组织，家庭成员互相关注老人健康

AI 增值：
  - 健康趋势图表（血压、血糖、体重长期变化）
  - 就诊前自动生成病历摘要（供医生快速了解）
  - 用药相互作用检查
  - 症状变化异常提醒
```

**竞争壁垒：** 医疗数据隐私敏感度极高，自部署是唯一可被用户信任的方案。现有健康 App 全部是云端，用户信任度低。

---

### 7. 投资交易日记

```
每笔交易 → 结构化 Markdown（入场逻辑、仓位、止损位、出场复盘）
Issues  → 待研究的标的、宏观风险事件追踪
Wiki    → 投资框架、行业研究、估值方法论
Actions → 持仓到目标价格自动提醒（结合行情 API）

AI 增值：
  - 决策模式分析（"你的亏损单 80% 发生在情绪化加仓后"）
  - 情绪偏差识别（FOMO 追高、恐慌割肉特征识别）
  - 绩效归因（超额收益来自哪类交易？）
  - 策略一致性检查（实际操作是否符合自己的框架？）
```

**竞争空白：** 交易日记工具几乎是空白市场，现有的都很简陋。做投资的人极度需要复盘工具，但没有人认真做。

---

### 8. 教育 / 课程管理

```
课程材料  → Markdown 文件，老师版本控制维护
作业提交  → 学生 Fork 仓库 + 提交 PR（天然的作业提交系统）
Issues   → 学生提问、班级公告、答疑
Wiki     → 课程大纲、参考资料、评分标准

AI 增值：
  - 作业自动初评（代码题自动测试，文章题评分参考）
  - 个性化学习路径推荐
  - 学生薄弱点识别（"这道题 70% 的学生做错了"）
  - 课程材料更新建议
```

---

### 9. 新闻 / 内容编辑 CMS

```
每篇文章  → Markdown + PR 编辑审稿流（记者写 → 编辑 Review → 主编 Merge）
Issues   → 选题库、采访任务分配、事实核查待办
Wiki     → 报道风格指南、采访资源库、禁用词表
Actions  → PR Merge 后自动发布到网站

AI 增值：
  - 标题 A/B 测试建议
  - 事实性陈述标记（需要核查的句子高亮）
  - SEO 关键词建议
  - 多语言自动翻译（初稿）
```

---

### 10. HR / OKR 管理

```
每季度 OKR → Markdown 文件，版本可追溯，历史可回顾
PR         → 绩效评估：员工自评 + 上级 Review，逐条评论
Issues     → 招聘需求、培训项目、政策变更
Wiki       → 公司政策、员工手册、薪酬结构

AI 增值：
  - OKR 质量评分（是否可量化、是否有挑战性）
  - 自动生成绩效评估初稿（基于 Issues 完成情况）
  - 招聘 JD 撰写辅助
  - 离职风险预警（基于绩效趋势）
```

## 执行策略

**统一底座，快速复制：**

所有产品共享：
- Gitea 实例（或用户自己的 GitHub）
- OpenMnemo 的心跳 + 同步基础设施
- 统一的 AI 翻译层（自然语言 → Gitea API）
- 统一的报告 / 可视化引擎

**差异只在：**
- 领域专属的 Markdown 模板（合同模板、病历模板、OKR 模板）
- 领域专属的 AI Prompt（法律条款分析、健康趋势分析）
- 领域专属的 UI（只是换壳）

**上市顺序建议（按痛点深度 + 市场空白）：**

```
P0（当前）：OpenMnemo — AI 记忆管理（已在做）
P1（下一个）：个人 CRM — 痛点明确，技术成本最低
P2：创意写作平台 — 有小说平台想法作为起点
P3：法律文书管理 — 痛点最深，单价最高
P4：学术研究管理 — 需要高校渠道
```
