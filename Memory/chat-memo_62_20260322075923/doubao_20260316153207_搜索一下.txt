XSS 检查是验证 Web 应用是否存在跨站脚本漏洞的安全测试过程，核心是检测用户输入是否被不当嵌入并执行恶意脚本，覆盖反射型、存储型、DOM 型三类场景，需结合手动验证、自动化扫描与防御加固形成闭环。
一、核心概念与风险
定义：XSS 允许攻击者向网页注入恶意客户端脚本（多为 JavaScript），在受害者浏览器执行，窃取 Cookie、篡改内容或冒充用户。
三大类型：
表格
类型	核心特征	常见场景	检测要点
反射型（非持久）	脚本通过 URL / 表单提交，服务器直接反射，单次有效	搜索框、错误页、URL 参数展示	检查 URL/POST 参数回显是否转义
存储型（持久）	脚本存入服务器（数据库 / 文件），所有访问者触发	评论、留言、用户资料	提交后退出重登，查看渲染结果
DOM 型	仅前端 DOM 操作注入，服务器无感知	前端路由、URL Hash 解析、本地存储读取	禁用 JS 后测试，对比前后端响应
主要危害：会话劫持、敏感信息泄露、网页篡改、钓鱼攻击、恶意代码执行。
二、手动检查（精准定位，必做）
1. 输入点全覆盖
重点测试所有外部输入入口：URL 查询参数、POST 表单、HTTP 头（User-Agent/Referer）、AJAX 请求、WebSocket 数据、本地存储（localStorage/sessionStorage）。
2. 分场景测试 Payload（核心用例）
表格
测试场景	基础 Payload	验证方式
HTML 上下文	<script>alert(document.domain)</script>	页面弹窗，源码可见未转义脚本
HTML 属性	" onmouseover="alert(1)" x="1	鼠标悬停触发弹窗
JavaScript 上下文	';alert(1);//	脚本执行，控制台无语法错误
URL 参数	?q=<img src=x onerror=alert(1)>	图片加载失败触发弹窗
DOM 型	#<script>alert(1)</script>	前端解析后执行，服务器响应无脚本
3. 验证步骤（标准化流程）
注入测试 Payload；
用浏览器 F12 查看响应源码，确认 Payload 是否未转义嵌入；
检查控制台是否有脚本执行痕迹；
存储型需退出重登或用另一账号访问验证；
DOM 型需禁用 JavaScript对比，确认漏洞在前端。
三、自动化工具检查（高效批量，推荐）
1. 主流工具选型（按场景）
表格
工具	类型	优势	适用场景	命令示例
OWASP ZAP	开源全栈扫描	免费、可集成 CI/CD、规则全面	常规 Web 应用、批量扫描	zap.sh -quickurl https://target.com -quickout report.html
Burp Suite（社区版）	代理 + 手动验证	灵活可控、插件丰富（XSS Validator）	精准测试、复杂场景绕过	拦截请求→发送到 Repeater→注入 Payload
DalFox	专项 XSS 扫描	轻量、高效、支持 DOM + 反射型	现代前端应用、快速验证	dalfox url https://target.com/?q=1 -b
XSStrike	AI 驱动 Payload	智能绕过过滤器、误报低	复杂过滤规则的应用	xsstrike -u https://target.com/?q=1
2. 关键操作要点
自动化扫描：先用 OWASP ZAP 做全量扫描，筛选 XSS 告警，重点核查高风险项；
手动复核：自动化工具对 DOM 型和复杂过滤场景易漏报，需结合 Burp Suite 手动验证；
CI/CD 集成：将 ZAP 或自定义脚本加入流水线，每次部署前自动扫描，阻断漏洞上线。
四、防御加固（检查的最终目标）
1. 核心原则：按输出场景编码，白名单过滤
表格
防御手段	具体措施	工具推荐
输出编码	HTML 编码（<→&lt;）、JS 编码、URL 编码，根据输出位置选择对应规则	OWASP Java Encoder、ESAPI
输入过滤	采用白名单机制，仅允许合法字符，拒绝</>/'/"等特殊字符	HTML Purifier、DOMPurify（富文本）
现代框架防护	禁用v-html/dangerouslySetInnerHTML，使用框架自动转义	Vue、React
安全头配置	启用 CSP（内容安全策略），限制脚本来源，禁止内联脚本	CSP Evaluator（工具）
会话保护	Cookie 设置HttpOnly+Secure+SameSite，防止会话劫持	服务器配置（Nginx/Apache）
2. 常见误区与避坑
❌ 仅依赖前端过滤：易被绕过，必须服务端为主；
❌ 手动写正则过滤：漏洞多，优先使用成熟库；
❌ 富文本允许所有 HTML：用 DOMPurify 净化，保留安全标签 / 属性；
❌ 忽略 DOM 型 XSS：前端需避免innerHTML/eval等危险 API。
五、检查清单（快速落地）
输入点梳理：列出所有外部输入，形成清单；
分类型测试：反射型→存储型→DOM 型，逐一验证；
工具扫描：OWASP ZAP 全量扫描 + Burp Suite 手动复核；
防御验证：检查输出编码、CSP 头、Cookie 属性；
回归测试：修复后重新扫描，确保无漏报 / 复现。
六、针对你的技术栈建议（Next.js + 表格前端）
服务端渲染（SSR）：使用 Next.js 的next/head和内置转义，避免手动拼接 HTML；
表格组件（TanStack Table/VTable）：确保单元格内容用textContent渲染，禁用 HTML 插入；
富文本需求：若有用户自定义内容，用 DOMPurify 净化后再展示；
API 防护：对所有接口输入做服务端验证，输出时按场景编码；
CI/CD 集成：加入 ZAP 扫描步骤，阻断含 XSS 漏洞的代码部署。
