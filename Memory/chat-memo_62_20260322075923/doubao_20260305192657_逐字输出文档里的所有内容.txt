AgentCrawl Plan8 商业计划书（全新重构）
版本：v8.0日期：2026-03-05定位：面向普通办公用户的 AI 原生协作平台范围：一人公司、中小团队（5-200 人）、数据驱动业务团队
1. 战略定义
AgentCrawl 的核心不是做新的开发者平台，而是把成熟的开源底座重组为普通人可直接使用的办公协作系统：
交互入口：OpenClaw in IM + OpenClaw Web
协作底座：Gitea
关键机制：Everything is a Repository
增长飞轮：AI探索 -> 技能固化 -> 工作流自动化
一句话：让团队在一个系统里完成项目、任务、审批、知识、素材与自动化，并保留可追溯历史。
2. 市场问题与机会
2.1 已验证痛点（来自历史方案验证）
工具碎片化：办公人员常同时使用 8-12 个工具。
协作低效：信息散落导致检索与对齐成本高。
版本失控：缺少统一的版本化协作底座。
数据主权弱：关键数据散落于第三方 SaaS。
2.2 市场规模（用于融资叙事）
TAM：$50B+（企业 AI 应用与协作软件大盘）
SAM：$5B（AI 协作与自动化细分）
SOM：$500M（初期聚焦中国与亚太中小团队）
注：上述数字沿用历史计划中的内部口径，用于融资阶段一致叙事。
2.3 直观成本对比
表格
方案	典型构成	参考成本
传统工具栈	Slack + Trello + Notion + Workspace	~$40 / 人 / 月
AgentCrawl 团队版	统一协作 + 插件 + 记忆 + 素材检索	~$19-49 / 人 / 月（SaaS 口径）
目标价值不是 “替代一个工具”，而是 “替代一组割裂工具 + 增加数据资产能力”。

4. 核心能力与护城河
4.1 一切皆仓库
插件是仓库：独立版本、独立部署、可回滚。
工作流是仓库：可审计、可复制、可运营。
知识与记忆是仓库：长期沉淀为团队资产。
4.2 OpenClaw 记忆落库 Gitea
个人记忆：openclaw-memory-{user_id}
项目记忆：task-{task_name}
组织知识：openclaw-shared-knowledge
结果：AI 的 “会话能力” 变成 “可检索、可审计、可传承” 的组织资产。
4.3 素材管理与多模态检索
文本：Meilisearch
图片 / 视频：CLIP/SigLIP + Qdrant
音频：Whisper 转录 + 全文索引
文件级定位：Everything/FSearch 适配层
价值：从 “采集即丢弃” 升级为 “可复用的数据资产池”。
4.4 三层信任与可逆执行（Plan8 增补）
探索层：OpenClaw，负责建议与意图解析。
命令层：CLI Gateway，作为写操作唯一入口。
工作流层：确定性执行引擎，节点级持久化。
核心约束：写操作可追溯、可回放、可回滚。
